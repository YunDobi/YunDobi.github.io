Interviewer Scheduler

Booking Interview simply with clean UI

Easy to book an Appointment

This project is my first React project, and created for easy booking an interview with the interviewer by making a scheduler, and storing each interviewer's available time in the backend server. If any person book an interview with the interviewer, fetch the data to the backend server and insert the person on their time. Also, the UI at that time will be changing.

In this project, I focus on how to make stateful react components and it was a great experience for familiar with React systems, like state and props. Even though it was not easy, I experted in React for the front-end now, and building the backend server with the node-js was the breaktime for me in this project. At this time, I do not have any knowledges of the database, so I used the backend server to store the data like a database. Automate Testing and UI StoryBook

Each UI was designed by the storybook and tested the functions of the UI. For example, press the button, and update the source correctly. After using the StoryBook, I can be more focused on the feature and design of the UI and spent less time testing and fixing the bug.

Moreover, not just storybook UI, I used many test frameworks. Mocha/Chai for the unit test and Cypress for end-to-end automated testing simulation. The first time of using the Cypress, it gave me a fresh shock. testing as a user/customer automate! That was a fun time, so I was interested more in automated testing. This project is one of the most used testing frameworks in my projects.
